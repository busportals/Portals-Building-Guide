The Task Debug Panel is designed to help you test your setup and implementation of interactive studio items.

{% hint style="warning" %}
In order for the Debug Panel to operate the task system must be turned on. This means you have to access it _before_ editing anything task related.
{% endhint %}

You can access the Task Debug Panel by going to Space Options > Task Debug Panel

<figure><img src="../../.gitbook/assets/image (12).png" alt="" width="144"><figcaption></figcaption></figure>

The task debug panel allows you to effortlessly switch between the states of tasks. The colored circles correspond to each state.

* Red - Not Active
* Yellow - Active
* Green - Completed

You can change the state of all single player or multiplayer task as well. The debug panel is especially helpful when you are trying to test out different effects.

Under each task you can also see the current task state and at the bottom of the menu you can see a complete history of task state changes.
